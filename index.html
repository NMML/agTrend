Installing agTrend

There are two basic methods to install the agTrend package, from within R itself using the devtools package and downloading a binary appropriate to your operating system. If you are using a windows machine it is possible that you will need Rtools installed to build and install the package. There is no compiled code in agTrend.

Using devtools

You can obtain the agTrend package using the GitHub installation function within the devtools R package. If devtools is not installed, it can be installed from CRAN by typing install.packages("devtools") from within the R console.

If devtools is installed, to install agTrend, open the R console and type

> library(devtools)
> install_github("agTrend", username="NMML")

Downloading a binary file

I have also placed copies of the binary installation files in a Google drive. Follow the following links for your operating system: Mac OSX or Windows. If you are running a Linux machine, see the following section. Note: make sure your browser does not try to automatically unpack the file (this is the default for Safari, go into preferences to change this feature).

After you download the file, open R and set the working directory (e.g., using setwd("path/to/download/dir/")). Once this is set, run

> install.packages("agTrend.zip", repos=NULL)

If you have a mac replace with "agTrend.zip" with "agTrend.tgz".

Downloading the source file Source files for each version can be obtained here. The desired file can be downloaded and installed in the same way as with the binaries, e.g., 

> install.packages("agTrend_0.11.tar.gz", repos=NULL, type="source")

Is it installed?

However you choose to install you should see some output that, hopefully, ends with:

* installing *source* package 'agTrend' ...
** R
** data
** demo
** inst
** preparing package for lazy loading
** help
*** installing help indices
** building package indices
** testing if installed package can be loaded
* DONE (agTrend)

In which case, everything is installed.
